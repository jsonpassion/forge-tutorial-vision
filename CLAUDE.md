## 콘텐츠 작성 규칙 (가장 중요!)

### 톤 & 스타일
- **언어**: 한국어 (기술 용어는 영문 병기. 예: "합성곱(Convolution)")
- **화자**: "~합니다" 존댓말 사용, 친근하지만 전문적인 톤
- **대상 독자**: 프로그래밍은 할 줄 알지만 CV는 처음인 개발자
- **핵심 원칙**: "초등학생도 비유로 이해할 수 있게, 하지만 대학원생도 깊이에 만족할 수 있게"
- **어투 가이드**: 딱딱한 교과서체가 아닌, **옆에서 설명해주는 선배** 느낌으로 작성
  - "~입니다" 일변도 대신 "~인데요", "~거든요", "~죠" 등 자연스러운 종결어미 적절히 혼용
  - 독자에게 질문을 던지는 구문 활용: "왜 그럴까요?", "어떤 차이가 있을까요?"
  - 놀라운 사실이나 반전을 만나면 감탄/공감 표현: "놀랍게도", "사실은", "의외로"
  - 단, 과도한 구어체나 반말은 금지. 존댓말 기반의 친근한 톤 유지

### 비유와 예시 규칙
- 모든 새로운 개념에는 반드시 **일상적 비유**를 먼저 제시
- 비유 → 실제 개념 → 수식(필요 시) → 코드 순서로 설명
- 비유 예시:
  - 픽셀 → "모자이크 타일 한 칸"
  - CNN 필터 → "돋보기로 이미지의 특정 패턴을 찾는 것"
  - 역전파 → "시험 오답노트를 만들어 실수를 줄여가는 과정"
  - Diffusion → "깨끗한 사진에 노이즈를 뿌렸다가 다시 복원하는 과정"
  - Attention → "시험 문제를 풀 때 중요한 부분에 형광펜을 치는 것"

### 스토리와 맥락 규칙

### 코드 블록 vs 마크다운 구분 (매우 중요!)

앱에서 코드 블록(` ``` `)은 "code" 라벨이 붙고 코드 전용 스타일로 렌더링됩니다.
**실행 가능한 코드만 코드 블록에 넣고, 다이어그램/시각적 설명은 반드시 일반 마크다운으로 작성합니다.**

- **코드 블록 사용 O**: Python/PyTorch 실행 코드, 터미널 명령어, 설치 명령
- **코드 블록 사용 X**: 개념 다이어그램, 구조 설명, 데이터 흐름도, 배열 시각화, ASCII art

다이어그램 대체 방법:
- **테이블**: 배열/행렬 구조, 비교표, 아키텍처 구조
- **인용(`>`)**: 부연 설명, 흐름 요약
- **볼드/이모지**: 채널 표시 (🔴 R, 🟢 G, 🔵 B), 단계 강조
- **리스트**: 순서가 있는 흐름/파이프라인

### 특수 코드 블록 태그

ForgeCore의 CodeBlockView는 기본 구문 하이라이팅 외에 특수 언어 태그를 지원합니다.
코드 실행 결과, 에러 메시지, 터미널 출력을 시각적으로 구분하여 표시할 수 있습니다.

| 태그 | 용도 | 스타일 |
|------|------|--------|
| `output`, `result` | 코드 실행 결과 | 회색 배경, 모노스페이스 |
| `error` | 에러 메시지 | 빨간 틴트 배경 |
| `console`, `terminal` | 터미널 출력 | 다크 배경, 초록 텍스트 |
| `run:{언어}` | 인터랙티브 실행 시뮬레이션 | 실행 버튼 + 출력 애니메이션 |

사용 예시:

#### 인터랙티브 실행 블록 (`run:`)

`run:{언어}` 태그를 사용하면 사용자가 "실행" 버튼을 눌러 코드 실행을 시뮬레이션할 수 있습니다.
바로 다음에 `output` 블록을 배치하면 예상 출력으로 자동 페어링됩니다.

````markdown
```run:python
name = "Vision"
print(f"안녕하세요, {name}!")
print("Computer Vision을 배워봅시다")
```

```output
안녕하세요, Vision!
Computer Vision을 배워봅시다
```
````

**사용 규칙:**
- `run:` 직후에 `output` 블록을 반드시 배치 (출력이 있는 코드의 경우)
- `print()` 등 텍스트 출력 코드만 사용 (GUI 코드, 이미지 출력 제외)
- 한 섹션에 1~3개 적정 (너무 많으면 학습 흐름 방해)
- 출력이 코드 실행 결과와 정확히 일치해야 함

- **코드 실행 결과**는 주석(`#`) 대신 `output` 또는 `result` 블록으로 분리하여 가독성 향상
- **에러 예시**는 `error` 블록으로 시각적으로 경고 표시
- **터미널/설치 명령의 출력**은 `console` 또는 `terminal` 블록 사용
- 간단한 한 줄 결과는 기존처럼 주석(`# [1, 16, 28, 28]`)으로 표시해도 무방

### 코드 예제 규칙
- **프레임워크**: Python + PyTorch 기준 (OpenCV는 전통 CV 파트)
- **실행 가능**: 복사-붙여넣기로 바로 실행할 수 있어야 함
- **주석**: 핵심 라인마다 한국어 주석 필수
- **출력 포함**: 코드 실행 결과를 주석이나 별도 블록으로 표시
- **점진적**: 간단한 예제 → 심화 예제 순서

```python
# 좋은 예시
import torch
import torch.nn as nn

# 3x3 합성곱 레이어 생성 (입력 채널 1, 출력 채널 16)
conv = nn.Conv2d(in_channels=1, out_channels=16, kernel_size=3, padding=1)

# 28x28 흑백 이미지 1장 생성 (배치, 채널, 높이, 너비)
x = torch.randn(1, 1, 28, 28)

# 합성곱 적용
output = conv(x)
print(f"입력 크기: {x.shape}")      # [1, 1, 28, 28]
print(f"출력 크기: {output.shape}")  # [1, 16, 28, 28]
```

- **프레임**: 테두리 + 미세 그림자

```json

  "chapters": [
    {
      "id": "chapter-id",
      "title": "챕터 제목",
      "description": "설명",
      "sections": [
        {
          "id": "section-id",
          "title": "섹션 제목",
          "file": "폴더/파일명.md",
          "description": "설명"
        }
      ]
    }
  ]
```

manifest.json 구조는 절대 임의로 변경하지 않습니다. 변경이 필요하면 사용자에게 먼저 확인합니다.

## 중요 주의사항
1. **순서 준수**: 반드시 manifest.json의 챕터/섹션 순서대로 작성
2. **일관성**: 모든 섹션이 동일한 구조와 톤을 유지
3. **웹 검색 필수**: 콘텐츠 작성 전 반드시 해당 주제의 최신 자료를 웹 검색
4. **status 업데이트 필수**: 섹션 완료 시 contents_status.md 반드시 갱신
5. **참고 자료 필수**: 모든 섹션에 유효한 외부 링크 포함
6. **코드 블록 구분**: 실행 코드만 코드 블록, 다이어그램/구조 설명은 마크다운 테이블/인용 사용
7. **스토리 필수**: 모든 섹션에 역사적 에피소드, 탄생 배경, 발명자 이야기 등 최소 1개 포함
8. **오해/팁 필수**: 모든 섹션에 흔한 오해, 혼동되는 개념, 실무 팁 중 최소 1개 포함
9. **친근한 어투**: 교과서체를 지양하고, 옆에서 설명해주는 선배 느낌의 자연스러운 톤 유지
